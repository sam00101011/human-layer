# Strategy

## Core strategic insight

If many websites adopt World ID, verification itself becomes a commodity.

That is good for Human Layer.

It means the company does not need to win by being the first app that says:
- this user is human

Instead, it can win by becoming the product that makes verified humanhood useful across the web.

## What stays differentiated

A website using World ID will usually only verify a user inside its own local context.

Human Layer can operate across sites and turn verification into:
- cross-site reputation
- human search
- human graph and follow graph
- cross-site interest discovery
- human-only feeds and rankings
- browser-level Human Mode
- portable trust and moderation signals

The shift is:

From:
- we verify humans

To:
- we organize the web around verified humans

## Where Human Layer can differentiate most

### 1. Cross-site reputation

Not just:
- verified on one app

But:
- high-signal verified human across GitHub, docs, Hacker News, Product Hunt, X, crypto, and research

This can include:
- expertise tags
- useful-rate
- verdict history
- endorsements
- moderation history
- contribution patterns

### 2. Human search

Rank:
- pages
- people
- products
- ideas

by verified-human judgment rather than bots, SEO, or engagement bait.

### 3. Human Graph

Help users find:
- other verified humans
- aligned interests
- trusted experts
- high-signal people across the web

This becomes the social layer above isolated World ID integrations.

### 4. Human Mode

Create a product experience websites themselves will not usually provide:
- hide ads
- hide bot-heavy comments
- show verified-human summaries, verdicts, and discussion
- clean reading mode with human context

### 5. Portable trust

Build signals that travel:
- verdicts
- summaries
- endorsements
- expertise
- trust scores
- moderation outcomes

Websites may verify users, but they usually will not build portable trust systems that work across the open web.

## First-mover advantage

The real first-mover advantage is not:
- we integrated World ID first

It is:
- we captured the best verified-human data earliest
- we built the strongest human graph first
- we created the first habit loops around verified-human search and discovery
- we became the default humanhood layer others build on

## How to use the first-mover advantage

### 1. Own the best initial surfaces

Start where signal quality matters and noise is painful:
- GitHub
- docs
- Hacker News
- Product Hunt
- YouTube
- crypto research, wallet, and token surfaces

These are the places where verified-human overlays and discovery feel obviously useful.

### 2. Capture unique data early

Build the dataset other companies will not have:
- page verdicts
- human summaries
- follows
- saves
- endorsements
- expertise tags
- reputation trails
- human-to-human trust edges

This dataset becomes a moat.

### 3. Define the schema

Standardize the primitives before the space gets crowded.

Important primitives:
- verdict types
- evidence model
- profile reputation fields
- expertise tags
- follow and endorsement mechanics
- human search ranking inputs

If Human Layer defines the useful structure early, later products may still rely on its model.

### 4. Build habit before competitors do

Create recurring behaviors:
- Suggested Humans
- human search
- topic feeds
- saved searches
- weekly digests of what verified humans found useful

The more often users return, the stronger the graph and reputation moats become.

### 5. Become infrastructure, not just an app

Offer:
- APIs
- widgets
- moderation tools
- trust overlays
- human discovery services
- anti-sybil access control
- reputation and jury systems

If websites adopt World ID directly, they may still use Human Layer because Human Layer owns the useful network and data layer on top.

## Strong strategic framing

World ID is the identity primitive.

Human Layer should become:
- the attention layer
- the trust layer
- the discovery layer

That is the durable strategic difference.

## Practical implication

The company should not frame itself as:
- another World ID app

It should frame itself as:
- the verified-human layer for the web
- the human graph for the internet
- the place where verified humanhood becomes useful, portable, and searchable
